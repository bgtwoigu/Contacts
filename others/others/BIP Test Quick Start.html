BIP Basic Introduction
 
    BIP(Bearer Independent Protocol)，是一种标准化的高速接口协议，通过该协议可以为SIM卡开启除SMS之外的新的数据传输信道，例如GPRS。通过BIP协议结合SIM卡应用（SAT），手机终端允许USIM卡和远程服务器之间进行透明数据传输，使得各种业务数据下载变得更加容易、快捷。例如，运营商可以向SIM卡发布内容和应用。
 
    BIP功能，是由一系列的BIP相关的SAT命令组合来实现的。BIP功能是由运营商来设计的，各个运营商的BIP功能设计不同。因此，BIP测试的目的只是确保BIP相关的命令或者Events能够被手机正确处理。至于运营商怎么组合这些命令去实现BIP功能(比如 SPN 的更新以及Menu item的变更)，只有运营商清楚。 如果您要了解运营商的BIP功能，需请运营商提供具体的功能介绍以及设计流程。

 

1.1 BIP测试相关的命令和Event

OPEN CHANNEL  ---- 要求手机能成功建立供BIP使用的PDP连接。也就是要建一个数据通道，用于SIM卡跟网络之间的数据交互。

SEND DATA ---- SIM卡通过手机发送数据给网络端。

RECEIVE DATA --- SIM卡通过手机接收从网络端发来的数据。

CLOSE CHANNEL ---- 要求手机关闭之前建立供BIP使用的数据通道。

SET UP EVENT LIST ---- 要求手机监听该STK命令中包含的 events。在BIP测试中，一般该命令在 OPEN CHANNEL 命令之前就已经上报了，且该命令中的 event list 中必须包含 data available 。

Data available event --- 手机监听到 data available 事件后，如果确定 SIM卡需要手机监听这个事件(即包含有 data available 的SET UP EVENT LIST命令已上报)，则会下发 envelope 来通知卡，卡就会知道网络端有数据发来。之后，卡会上报 RECEIVE DATA 命令来接收网络端发来的数据。

 

1.2 BIP的主要流程

      SET UP EVENT LIST à OPEN CHANNEL à SEND DATA à data available event以及对应的 envelope  à RECEIVE DATA à CLOSE CHANNEL

其中：

SEND DATA à data available event以及对应的 envelope  à RECEIVE DATA：  这个过程可能会重复多次。当数据通道打开后，SIM卡和网络端的数据交互可能会有多次。

RECEIVE DATA ： 也可能重复多次。当数据太大，SIM卡可能会多次上报该命令来分批接收数据。

OPEN CHANNEL à SEND DATA à data available event以及对应的 envelope  à RECEIVE DATA à CLOSE CHANNEL ：这个过程也可能会重复多次。SIM卡可能循环多次去要求手机打开一个数据通道，然后数据交互后再关闭该通道。之后多次重复这个过程。

 

Q&A

(1). MTK平台对SAT版本的支持情况

[Answer]

目前 MTK平台支持的是 R99 和 R4.

(2). KK上MTK_BIP_SCWS这个宏的作用是什么？运营商处过BIP测试时是否需要开启？开启后会不会对之前的BIP测试有影响？

[Answer]

MTK_BIP_SCWS这个宏是用来控制 BIP(Bearer Independent Protocl) for SCWS(Smart Card Web Service) 这个feature 的开关的。

运营商处过BIP测试时需要开启。

该宏 打开后，主要是BIP方面会有影响。对于 OPEN CHANNEL 、CLOSE CHANNEL 、GET CHANNEL STATUS等跟数据连接有关的STK 的命令都会有影响，比如调用 establishLink 建立连接时，可以有 TCP link 和 TCPServer link。即该宏打开后，可以支持基于 TCP 连接的传输。没打开这个宏之前，只支持基于 UDP连接的传输。具体的 code 在 BipManager.java、BipChannelManager.java、Channel.java这三个文件中，这三个都是跟 BIP 有关的文件。

该宏 打开后，不会对之前的BIP 测试产生影响。

 

(3). BIP protocol是否支持 BIP over CAT-TP？

[Answer]

BIP over TCP(也即通过TCP连接来传输数据)，要SCWS功能开启了才支持。下面针对 BIP over TCP 部分的各个 command 和 event 的支持情况都是在 SCWS功能开启的前提下的支持情况：

(1). 对于如下 proactive commands:

  Proactive commands          BIP over CAT_TP(即over UDP)      BIP over TCP

o OPEN CHANNEL                          Y                                                         Y

o CLOSE CHANNEL                         Y                                                         Y

o SEND DATA                                  Y                                                         Y

o RECEIVE DATA                             Y                                                         Y

o GET CHANNEL STATUS              Y                                                          Y

 

(2). 对于如下 Events：

  Events                                             BIP over CAT_TP(即over UDP)      BIP over TCP

o DATA AVAILABLE                                                      Y                                      Y

o CHANNEL STATUS OTA (SMS – BIP CATTP)          N                                     N
