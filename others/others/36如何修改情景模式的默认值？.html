何修改情景模式的默认值？

UI介绍：

1，  情景模式下菜单下分预定义的几种情景模式(标准/静音/会议/户外)

2，  每个情景模式点进去，分几大类:常规/音量/来电/通知/系统设置

 
[SOLUTION]

一,针对每个情景模式的默认值修改：

 

1,默认铃声是统一在alps/build/target/product/core.mk文件中设置，其中ro.config.notification_sound表示通知的默认铃声，ro.config.ringtone表示语音来电和视频来电的铃声，若希望语音来电和视频来电分开设置，请参考FAQ08273 [Audio Profile]如何设置视频来电铃的默认铃声:

PRODUCT_PROPERTY_OVERRIDES:=\

ro.config.notification_sound=Proxima.ogg\

ro.config.alarm_alert=Alarm_Classic.ogg\

ro.config.ringtone=Backroad.ogg

 

2,情景模式中的其他设定参考到alps/mediate/framework/base/media/audio/java/com/mediate/audioprofile/AudioPrifileMnager.java中的getDefaultState()

 

1）上图中红色框标所示表示是否有IDefaultProfileStatesGetter接口的实现，若没有，则默认设置在getDeafultSte中绿色框中配置，其中DEFAULT_RINGER_VOLUME_GENERAL=4,表General Profile的默认的铃声音量，以此类推，其他的默认值也是如此命名；

 

2）若能在MediatekClassFactory.java或者MediatekOPClassFactory.java中搜到IDefaultProfileStatesGetter的实现类，如TDD项目对应的IDefaultProfileStatesGetter的实现类是DefaultProfileStatesGetterOP01.java,则情景模式的默认值要修改alps/mediate/operator/op01/frameworks/java/com/mediate/op/audioprofileDefaultProfileStatesGetterOP01.java的getDefaultPofileStates()：

 

 

3,在AudioManager.java

DEFAULT_STREAM_VOLUME [] 数组中请把对应的stream type 音量值修改一致。比如把来电铃声从默认4调到6
4,  // STREAM_RING

改为

6, // STREAM_RING

 

二，download bin到手机之后如果从来都没有进入到情景模式菜单修改过任何一个情景模式，则情景模式就不会synchronize 自己的设定到系统设定，因此这种情况若希望系统的默认设定和情景模式默认Active的（默认Active的是General Profile）的设定一致的话，可以修改如下系统的默认值来与前面所说的的情景模式的具体值相匹配：

 

alps/frameworks/base/packages/settingsprovider/res/valuse/Defaults.xml:

<bool name="def_haptic_feedback">true</bool>

<integer name="def_lockscreen_sound_enabled">1</integer>

<bool name="def_sound_effects_enabled">false</bool>

<bool name="def_dtmftone_enabled">true</bool>

如默认的active的情景模式是General，假设在前面设置了DEFAULT_DTMFTONE_GENERAL = false，则对应的系统的默认的dtmf如果要保持与情景模式一致，则需要设置def_dtmftone_enabled对应为true，若DEFAULT_SOUNDEFFECT_GENERAL=false，则对应的系统默认的sound effect的值如果保持与情景模式一致，则需要设置def_sound_effects_enabled为false。以此类推其他的设置。
