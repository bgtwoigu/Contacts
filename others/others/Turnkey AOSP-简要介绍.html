在 Android L 版本，Mediatek 将对 Turnkey Solution 做一次 Migration，即从 Turnkey ABS 架构演变为 Turnkey AOSP。并且将全面转向 Turnkey AOSP 架构，不再使用现有的 Turnkey ABS 架构。

Turnkey AOSP 架构相对于 Turnkey ABS 架构主要做了如下几个方面的改动：

  • 目录结构。采用 AOSP 建议的摆放方式，放弃之前 Turnkey ABS 架构中 Mediatek 集中式的摆放。
  • 编译系统。采用 AOSP 默认的编译系统，将 Mediatek 的部分做整合。
  • Kernel 标准化。Kernel code 完全独立摆放，编译方式和客制化方式都采用开源 Linux Kernel 的标准方式。Kernel API 也在逐渐推进采用 Kernel 标准的命名方式。
  • Java API 的变化。

但 Turnkey AOSP 依然是 Turnkey Solution，我们的 Turnkey Feature 都会 Porting 到 Turnkey AOSP 架构上面去。为了能够让大家在 L 版本到来的时候能够快速上手开发，我们在 KitKat 版本上面开发了预览版供大家学习使用。并且对所有相关的开发文档都做了更新以支持 Turnkey AOSP 架构的开发。
